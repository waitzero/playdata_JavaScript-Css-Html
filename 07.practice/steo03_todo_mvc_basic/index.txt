Step02에서의 한계점
1. 여전히 App.java 한 곳에서 다양한 역할을 수행하고 있음 -> GodClass
	1) 데이터의 입출력 및 파싱(parsing) 처리
	2) 데이터의 출력 결과를 보여주는 View 기능
	3) 어떤 데이터를 조회할지 등을 결정하는 Controller 기능 등등
	
Step03
1. 읽어들인 데이터를 parsing해주는 코드를 별도의 클래스 TodoTSVParser.java로 분리
2. 분리한 클래스가 위치한 패키지명은 dev.todo.data, data접근 역할에 해당하는 부분
3. App.java에서 조회에 해당하는 부분(View 기능)들도 별도의 메서드로 분리

개선된 부분
1. 데이터를 파싱하는 부분을 별도의 클래스로 관심사(역할)를 분리함으로써 관리가 용이해짐

현재 한계점
1. 데이터를 파싱하는 부분을 별도의 클래스 분리하였지만,
여전히 메인 화면에 해당하는 App.java에서 View, 연산 처리 등을 전부 한 곳에서 수행 중
